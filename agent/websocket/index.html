WebSocket Protocol

에이전트와 서버 간의 실시간 통신은 Socket.io를 통해 이루어집니다.

연결

Section titled “연결”

서버는 연결 시 isAgent 플래그를 확인하여 에이전트 소켓을 별도 추적합니다.

이벤트

Section titled “이벤트”

agent:task (서버 → 에이전트)

Section titled “agent:task (서버 → 에이전트)”

이슈가 에이전트에게 할당되면 자동 전송됩니다.

agent:progress (에이전트 → 서버)

Section titled “agent:progress (에이전트 → 서버)”

작업 진행률을 실시간으로 보고합니다.

UI에 실시간 프로그레스 바로 표시됩니다.

agent:done (에이전트 → 서버)

Section titled “agent:done (에이전트 → 서버)”

작업 완료를 보고합니다.

agent:heartbeat (에이전트 → 서버)

Section titled “agent:heartbeat (에이전트 → 서버)”

30초마다 전송하여 연결 상태를 확인합니다.

인증

Section titled “인증”

API Key (ipl_ prefix) 또는 JWT 모두 지원됩니다.

폴링 Fallback

Section titled “폴링 Fallback”

WebSocket 연결이 끊기면 자동으로 폴링 모드로 전환됩니다 (30초 간격).
